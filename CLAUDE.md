## Project Overview

ContextFlow (Local-First AI Tab Manager)

## Core Philosophy

- Context over Folders: Users organize work by "Contexts" (Projects).
- Intelligence by Default: Automatic indexing and semantic search.
- Privacy First: 100% local processing; no external servers for data or AI.
- Performance: Zero UI lag; heavy tasks offloaded to Web Workers.

## Technical Stack

- Framework: React + Vite + Tailwind CSS.
- Manifest: Chrome Extension Manifest V3.
- Database: Dexie.js (IndexedDB wrapper).
- AI Layer: Transformers.js (@xenova/transformers) - Local Embeddings.
- API Focus: chrome.sidePanel, chrome.tabs, chrome.windows, chrome.scripting.

## Data Schema (Dexie)

- `tabs`: { id, url, title, favIcon, projectId, windowId, lastAccessed }
- `projects`: { id, name, color, lastOpened, isArchived }
- `embeddings`: { url, vector, snippet, timestamp } - For Semantic Search.
- `snapshots`: { id, timestamp, data } - For Global Browser state recovery.

## Power Features Logic

1. Unified Sidebar: Real-time list of all tabs across all windows.
2. The Great Merge: Consolidation of all windows into one.
3. The Duplicate Killer: Auto-closing redundant URLs.
4. Semantic Search: Using local AI to find tabs by "Meaning" (e.g., "That JS documentation from Tuesday").
5. Local Cloud Bridge: Auto-export JSON to a local Dropbox/Drive-synced folder via File System Access API.

## Architecture

ContextFlow operates under Chrome's Manifest V3 constraints, requiring a strict separation of concerns to handle AI models and background tasks efficiently.

- **Side Panel (`src/sidepanel/`)**: The main React application UI. It acts as the "view" layer, reading state primarily from Dexie.js and interacting with `chrome.tabs`/`chrome.windows` APIs.
- **Background Service Worker (`src/background/`)**: The event orchestrator. Listens for tab updates, creates snapshots, and triggers deduplication/merging. **Crucial:** It is fully stateless, as MV3 service workers can be terminated after 30 seconds of inactivity.
- **Content Scripts (`src/content/`)**: Injected into web pages to extract page context (H1, Meta Description, main text) when a tab is fully loaded. Sends this text to the background script.
- **Offscreen Document (`src/offscreen/`)**: The AI Engine. Because Chrome MV3 strict Content Security Policies (CSP) often block WebAssembly (WASM) inside background Service Workers, Transformers.js runs inside an invisible Offscreen Document. This document receives text, generates vector embeddings, and saves them to Dexie.
- **Storage Layer**: 
  - **Dexie (IndexedDB)**: Single source of truth for heavy relational data (Tabs, Projects, Embeddings).
  - **Chrome Storage Local**: For lightweight user preferences (theme, sync folder paths).

## Important Conventions

- **MV3 Lifecycle Strictness**: Never use global variables in the Background Service Worker to store state. Always rely on Dexie.js or `chrome.storage`.
- **The "Optimistic Delete"**: UI must feel instant. When a user closes a tab or project via the UI, remove it from the React state immediately *before* awaiting the `chrome.tabs.remove()` or Dexie delete operations.
- **AI Singleton & Caching**: The Transformers.js model (`all-MiniLM-L6-v2`) must be initialized using a Singleton pattern inside the Offscreen Document to prevent downloading/loading the 30MB model multiple times.
- **React Performance**: Use `React.memo()` extensively on tab list items. A user might have 200 tabs open; updating the state of one tab should not re-render the entire list.
- **Message Passing Safety**: All `chrome.runtime.sendMessage` calls between the Side Panel, Background Worker, and Offscreen Document must use strict TypeScript interfaces to prevent silent failures.
- **Vector Storage Optimization**: To keep IndexedDB sizes manageable, ensure vectors generated by Transformers.js are quantized or stored efficiently (e.g., using `Float32Array` buffers rather than standard JS arrays) when saving to Dexie.
- **Saved Context Behavior**: When saving a context, all tabs are saved to the database with `windowId = -1`, then a new tab is created and all other tabs are closed. This keeps the browser alive while clearing the workspace. Saved tabs have `visitedAt` timestamps for date-based searching.
- **Minimal UI Design**: Keep the interface clean and distraction-free. Action buttons should be small icons placed inline with relevant content (e.g., window headers) rather than in large separate sections.
- **Multi-Select Pattern**: Use checkbox selection mode for bulk operations. When enabled, clicking a tab toggles selection instead of activating it. Show a "Close Selected" button when tabs are selected.
- **Date-Based Search**: The system supports natural language date queries like "last week", "yesterday", "3 days ago". These search saved tabs (windowId = -1) using the `visitedAt` timestamp. The search icon changes to a calendar when date terms are detected.
